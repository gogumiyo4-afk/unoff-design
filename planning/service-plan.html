이게 뭔가요?

사용자가 UNOFF에 처음 들어올 때 계정을 만들고, 다음에 다시 올 때 자동으로 로그인되게 해주는 기능이에요.

가입 방법은?

카카오/Apple 소셜 로그인 또는 연락처(SMS) 인증번호로 가입해요. 비밀번호 없이 전화번호 OTP만으로 로그인할 수 있어서 가장 간편해요.

왜 이렇게 만드나요?

비밀번호를 없앴어요. 소셜 로그인은 터치 2~3번, 연락처 로그인은 전화번호 입력 → 인증번호 입력이면 끝이에요. 20~35세 청년에게 익숙한 방식(당근, 토스)이고, 비밀번호 분실 문의도 없어져요.

보안은?

소셜 계정은 카카오/Apple이 직접 인증하고, 연락처 OTP는 매번 새 일회용 코드를 발송해서 안전해요. 개인정보는 탈퇴 시 30일 내 완전 삭제하고, 개인정보보호법을 준수해요.

  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,

  -- 기본 프로필
  name TEXT NOT NULL DEFAULT '',
  phone TEXT UNIQUE,                 -- 연락처 OTP 가입 시 저장
  email TEXT UNIQUE,                 -- 소셜 가입 시 저장 (nullable)
  nickname TEXT UNIQUE NOT NULL DEFAULT 'OFF 중인 누군가',
  bio TEXT DEFAULT '' CHECK (char_length(bio) <= 50),
  birth_date DATE NOT NULL,

  -- ON/OFF 상태
  status TEXT NOT NULL DEFAULT 'off'
    CHECK (status IN ('off', 'slow', 'on', 'pause')),

  -- 온보딩 서베이
  gap_duration TEXT CHECK (gap_duration IN (
    'under_3m', '3m_to_1y', 'over_1y', 'burnout', 'browsing', NULL)),
  gap_reason TEXT CHECK (gap_reason IN (
    'job_search', 'resign', 'health', 'voluntary', 'hard_to_say', NULL)),
  interests TEXT[] DEFAULT '{}',

  -- 배터리
  battery_percent INTEGER NOT NULL DEFAULT 0 CHECK (battery_percent BETWEEN 0 AND 100),
  battery_level INTEGER NOT NULL DEFAULT 1 CHECK (battery_level BETWEEN 1 AND 5),

  -- 인증
  auth_provider TEXT NOT NULL DEFAULT 'phone'
    CHECK (auth_provider IN ('phone', 'kakao', 'naver', 'apple')),
  is_school_verified BOOLEAN DEFAULT FALSE,

  -- 알림
  push_enabled BOOLEAN DEFAULT FALSE,
  notification_settings JSONB DEFAULT '{"comments":true,"reactions":true,"together":true,
    "matching":true,"battery":true,"notice":true,"quiet_start":"23:00","quiet_end":"08:00"}',

  -- 약관 동의 (법적 증거)
  terms_agreed_at TIMESTAMPTZ,
  privacy_agreed_at TIMESTAMPTZ,
  marketing_agreed BOOLEAN DEFAULT FALSE,
  marketing_agreed_at TIMESTAMPTZ,

  -- 메타
  onboarding_completed BOOLEAN DEFAULT FALSE,
  last_active_at TIMESTAMPTZ DEFAULT NOW(),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  deleted_at TIMESTAMPTZ DEFAULT NULL
);

RLS 정책 백엔드

원칙: 모든 테이블 RLS 활성화. anon 키는 클라이언트 노출 OK (RLS 보호). service_role은 서버(Edge Function)에서만. 
-- users RLS
ALTER TABLE public.users ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users_select_own" ON public.users
  FOR SELECT USING (auth.uid() = id);

CREATE POLICY "users_select_public" ON public.users
  FOR SELECT USING (
    deleted_at IS NULL
    AND id NOT IN (SELECT blocked_id FROM blocks WHERE blocker_id = auth.uid())
  );

CREATE POLICY "users_update_own" ON public.users
  FOR UPDATE USING (auth.uid() = id) WITH CHECK (auth.uid() = id);

CREATE POLICY "users_insert_none" ON public.users
  FOR INSERT WITH CHECK (false);  -- Edge Function(service_role)만 INSERT

CREATE POLICY "users_delete_none" ON public.users
  FOR DELETE USING (false);  -- soft delete만 허용

-- waitlist RLS
ALTER TABLE public.waitlist ENABLE ROW LEVEL SECURITY;
CREATE POLICY "waitlist_insert_anon" ON public.waitlist FOR INSERT WITH CHECK (true);
CREATE POLICY "waitlist_select_none" ON public.waitlist FOR SELECT USING (false);

// POST /functions/v1/auth-social
// Body: { provider: 'kakao' | 'naver', token: string }

async function handler(req) {
  const { provider, token } = await req.json()

  // 1. 소셜 API로 유저 정보 조회
  const userInfo = provider === 'kakao'
    ? await fetch('https://kapi.kakao.com/v2/user/me', {
        headers: { Authorization: `Bearer ${token}` }
      }).then(r => r.json())
    : await fetch('https://openapi.naver.com/v1/nid/me', {
        headers: { Authorization: `Bearer ${token}` }
      }).then(r => r.json())

  const { email, name } = userInfo

  // 2. 기존 유저 확인
  const { data: existing } = await supabaseAdmin.auth.admin.getUserByEmail(email)

  if (existing) {
    const existingProvider = existing.app_metadata?.provider
    if (existingProvider && existingProvider !== provider) {
      return error(409, {
        message: `이 이메일은 ${existingProvider}로 가입되어 있어`,
        existing_provider: existingProvider
      })
    }
    // 로그인
    const { data: session } = await supabaseAdmin.auth.admin.generateLink({
      type: 'magiclink', email
    })
    return ok({ session, is_new: false })
  }

  // 3. 신규 가입
  const { data: newUser } = await supabaseAdmin.auth.admin.createUser({
    email, email_confirm: true, app_metadata: { provider }
  })

  await supabaseAdmin.from('users').insert({
    id: newUser.id, email,
    nickname: name || 'OFF 중인 누군가',
    auth_provider: provider,
    birth_date: '2000-01-01',  // 온보딩에서 재입력
    terms_agreed_at: new Date().toISOString(),
    privacy_agreed_at: new Date().toISOString(),
  })

  const { data: session } = await supabaseAdmin.auth.admin.generateLink({
    type: 'magiclink', email
  })
  return ok({ session, is_new: true })
}

이게 뭔가요?

처음 가입한 사용자에게 7가지 간단한 질문을 하는 화면이에요. 나이, 지금 상태, 관심 분야 같은 걸 물어봐요.

왜 필요한가요?

비슷한 상태의 사람끼리 연결해주고, 맞춤 콘텐츠를 보여주려면 기본 정보가 필요해요. 하지만 모든 질문은 건너뛸 수 있어서 부담이 없어요.

어떤 느낌인가요?

"편하게 말해줘" 톤. 취조가 아니라 대화예요. 답하기 싫으면 건너뛰면 되고, 나중에 프로필에서 바꿀 수 있어요.

만 20~35세만?

네. 범위 밖이면 "곧 더 넓혀갈 거야"라고 안내하고, 대기 리스트에 이메일을 등록할 수 있어요. 거절이 아니라 대기예요.

  • 뒤로 가기 가능 (이전 단계로 돌아가기, 입력값 유지)

DB 컬럼: public.users.moon_phase INTEGER DEFAULT 0 (0~9, 10단계)
닉네임 변경 불가 정책: 닉네임은 가입 시 1회 설정 후 변경 불가. 운영팀에 사유와 함께 별도 요청 시에만 가능.
