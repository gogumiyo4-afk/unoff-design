1
유저가 "카카오로 시작" 버튼 탭프론트엔드 — LoginScreen
2
카카오 SDK가 인앱 브라우저로 카카오 로그인 화면 표시프론트엔드 — 카카오 SDK
3
유저가 카카오 계정으로 동의 → 카카오 액세스 토큰 반환프론트엔드 — 카카오 SDK callback
4
앱이 카카오 토큰을 Supabase Edge Function으로 전송프론트엔드 → 백엔드 — POST /functions/v1/auth-social
5
Edge Function이 카카오 API로 유저 정보 조회 (이메일, 닉네임) 백엔드 — Edge Function auth-social
6
Supabase Auth에서 해당 이메일로 유저 존재 확인 백엔드 — supabase.auth.admin.getUserByEmail()
7
분기: 기존 유저면 JWT 발급 / 신규면 자동 가입 + JWT 발급 백엔드 — supabase.auth.admin.createUser() 또는 generateLink()
8
JWT를 앱으로 반환 → SecureStore에 저장프론트엔드 — expo-secure-store
9
신규 유저 → OnboardingSurvey / 기존 유저 → HomeScreen 프론트엔드 — onboarding_completed 필드로 분기
네이버 로그인도 동일한 플로우 — SDK와 토큰 검증 API 엔드포인트만 다름.
Apple 로그인은 Supabase Auth가 기본 지원하므로 Edge Function 불필요. supabase.auth.signInWithIdToken({ provider: 'apple', token }) 사용.

DB 스키마 백엔드

auth.users (Supabase Auth 내장)

Supabase Auth가 자동으로 관리하는 테이블. 직접 수정하지 않음.
  • id — UUID, PK
  • email — 이메일
  • encrypted_password — 암호화된 비밀번호 (이메일 가입 시)
  • raw_app_meta_data — provider 정보 등
  • created_at, updated_at

public.users (앱 프로필 테이블)

CREATE TABLE public.users (
  -- PK: Supabase Auth의 auth.users.id와 동일
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,

  -- 기본 프로필
  email TEXT UNIQUE NOT NULL,
  nickname TEXT UNIQUE NOT NULL DEFAULT 'OFF 중인 누군가',
  bio TEXT DEFAULT '' CHECK (char_length(bio) <= 50),
  birth_date DATE NOT NULL,

  -- ON/OFF 상태
  status TEXT NOT NULL DEFAULT 'off'
    CHECK (status IN ('off', 'slow', 'on', 'pause')),

  -- 온보딩 서베이 결과
  gap_duration TEXT CHECK (gap_duration IN (
    'under_3m', '3m_to_1y', 'over_1y', 'burnout', 'browsing', NULL
  )),
  gap_reason TEXT CHECK (gap_reason IN (
    'job_search', 'resign', 'health', 'voluntary', 'hard_to_say', NULL
  )),
  interests TEXT[] DEFAULT '{}',

  -- 배터리 시스템
  battery_percent INTEGER NOT NULL DEFAULT 0
    CHECK (battery_percent BETWEEN 0 AND 100),
  battery_level INTEGER NOT NULL DEFAULT 1
    CHECK (battery_level BETWEEN 1 AND 5),

  -- 인증 관련
  auth_provider TEXT NOT NULL DEFAULT 'email'
    CHECK (auth_provider IN ('email', 'kakao', 'naver', 'apple')),
  is_school_verified BOOLEAN DEFAULT FALSE,

  -- 알림 설정
  push_enabled BOOLEAN DEFAULT FALSE,
  notification_settings JSONB DEFAULT '{
    "comments": true,
    "reactions": true,
    "together": true,
    "matching": true,
    "battery": true,
    "notice": true,
    "quiet_start": "23:00",
    "quiet_end": "08:00"
  }',

  -- 약관 동의
  terms_agreed_at TIMESTAMPTZ,           -- 이용약관 동의 시각
  privacy_agreed_at TIMESTAMPTZ,         -- 개인정보처리방침 동의 시각
  marketing_agreed BOOLEAN DEFAULT FALSE, -- 마케팅 수신 동의
  marketing_agreed_at TIMESTAMPTZ,       -- 마케팅 수신 동의 시각

  -- 메타
  onboarding_completed BOOLEAN DEFAULT FALSE,
  last_active_at TIMESTAMPTZ DEFAULT NOW(),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  deleted_at TIMESTAMPTZ DEFAULT NULL     -- soft delete
);
기존 스키마 대비 추가 항목
  • terms_agreed_at — 이용약관 동의 시각 (법적 증거용, 동의 철회 시 NULL)
  • privacy_agreed_at — 개인정보처리방침 동의 시각
  • marketing_agreed + marketing_agreed_at — 마케팅 수신 동의
  • id가 auth.users(id)를 참조 — Supabase Auth와 1:1 연동

RLS 정책 (Row Level Security) 백엔드

원칙: 모든 테이블에 RLS 활성화. anon 키는 RLS로 보호되므로 클라이언트 노출 가능. service_role 키는 RLS를 우회하므로 서버(Edge Function)에서만 사용.

public.users 테이블 RLS

-- RLS 활성화
ALTER TABLE public.users ENABLE ROW LEVEL SECURITY;

-- 본인 프로필 읽기
CREATE POLICY "users_select_own" ON public.users
  FOR SELECT USING (auth.uid() = id);

-- 다른 유저의 공개 프로필 읽기 (삭제되지 않은 유저만)
CREATE POLICY "users_select_public" ON public.users
  FOR SELECT USING (
    deleted_at IS NULL
    AND id NOT IN (
      SELECT blocked_id FROM blocks WHERE blocker_id = auth.uid()
    )
  );

-- 본인 프로필 수정
CREATE POLICY "users_update_own" ON public.users
  FOR UPDATE USING (auth.uid() = id)
  WITH CHECK (auth.uid() = id);

-- 프로필 생성 (가입 시 Edge Function에서 service_role로 수행)
-- 클라이언트에서 직접 INSERT 불가
CREATE POLICY "users_insert_none" ON public.users
  FOR INSERT WITH CHECK (false);

-- 삭제 (탈퇴는 Edge Function에서 soft delete로 처리)
CREATE POLICY "users_delete_none" ON public.users
  FOR DELETE USING (false);

public.waitlist 테이블 RLS

ALTER TABLE public.waitlist ENABLE ROW LEVEL SECURITY;

-- 대기 등록은 누구나 가능 (가입 전이므로 anon)
CREATE POLICY "waitlist_insert_anon" ON public.waitlist
  FOR INSERT WITH CHECK (true);

-- 읽기/수정/삭제는 서버만 (service_role)
CREATE POLICY "waitlist_select_none" ON public.waitlist
  FOR SELECT USING (false);

Edge Functions 상세 백엔드

auth-social (소셜 로그인)

// POST /functions/v1/auth-social
// Body: { provider: 'kakao' | 'naver', token: string }

async function handler(req) {
  const { provider, token } = await req.json()

  // 1. 소셜 API로 유저 정보 조회
  let userInfo
  if (provider === 'kakao') {
    userInfo = await fetchKakaoUser(token)
    // GET https://kapi.kakao.com/v2/user/me
    // Headers: Authorization: Bearer {token}
  } else if (provider === 'naver') {
    userInfo = await fetchNaverUser(token)
    // GET https://openapi.naver.com/v1/nid/me
    // Headers: Authorization: Bearer {token}
  }

  const { email, name, profileImage } = userInfo

  // 2. 기존 유저 확인
  const { data: existingUser } = await supabaseAdmin
    .auth.admin.getUserByEmail(email)

  if (existingUser) {
    // 2a. 다른 provider로 가입한 경우 → 에러
    const existingProvider = existingUser.app_metadata?.provider
    if (existingProvider && existingProvider !== provider) {
      return error(409, {
        message: `이 이메일은 ${existingProvider}로 가입되어 있어`,
        existing_provider: existingProvider
      })
    }

    // 2b. 같은 provider → 로그인 (JWT 발급)
    const { data: session } = await supabaseAdmin
      .auth.admin.generateLink({
        type: 'magiclink', email
      })
    return ok({ session, is_new: false })
  }

  // 3. 신규 유저 → 자동 가입
  const { data: newUser } = await supabaseAdmin
    .auth.admin.createUser({
      email,
      email_confirm: true,  // 소셜은 이메일 인증 불필요
      app_metadata: { provider }
    })

  // 4. public.users 프로필 생성
  await supabaseAdmin.from('users').insert({
    id: newUser.id,
    email,
    nickname: name || 'OFF 중인 누군가',
    auth_provider: provider,
    birth_date: '2000-01-01',  // 온보딩에서 입력받을 예정
    terms_agreed_at: new Date().toISOString(),
    privacy_agreed_at: new Date().toISOString(),
  })

  // 5. JWT 발급
  const { data: session } = await supabaseAdmin
    .auth.admin.generateLink({
      type: 'magiclink', email
    })

  return ok({ session, is_new: true })
}

  • 소셜 토큰은 반드시 해당 소셜 API로 검증 (클라이언트가 보낸 토큰을 그대로 신뢰하지 않음)
  • Rate Limiting: IP당 분당 10회 제한 (brute force 방지)

// App.tsx 또는 AuthProvider
const checkSession = async () => {
  const { data: { session }, error } = await supabase.auth.getSession()

  if (session) {
    // access_token 유효 → 홈 이동
    // 또는 refresh로 자동 갱신됨
    navigateTo('Home')
  } else {
    // 세션 없음 → 로그인 화면
    navigateTo('Login')
  }
}

// 세션 변경 감지 (실시간)

3

4
30일 유예 후 완전 삭제 (pg_cron 스케줄)유예 기간 내 재가입 문의 시 복구 가능

마케팅 동의 이력동의 철회 후 6개월법적 분쟁 대비 보관 후 삭제
법적 요구사항
  • 개인정보보호법 제21조 — 목적 달성 시 지체 없이 파기
  • 정보통신망법 — 이용자 동의 철회 요청 시 5일 이내 처리
  • 만 14세 미만은 법정대리인 동의 필요 — UNOFF는 만 20세 이상이므로 해당 없음
  • 마케팅 수신 동의/철회 이력은 반드시 시각(timestamp)과 함께 기록

엣지케이스 대응 FE + BE

상황대응담당
카카오로 가입 → 같은 이메일로 네이버 로그인 시도 "이 이메일은 카카오로 가입되어 있어. 카카오로 로그인해줘" + 카카오 버튼 하이라이트 BE (Edge Function) + FE
Apple "이메일 숨기기" 선택 시 Apple이 제공하는 relay 이메일을 그대로 사용. 유저에게 안내: "Apple 이메일로 가입했어" FE + BE
온보딩 중 앱 종료 onboarding_completed = false → 다음 로그인 시 이어서 진행. 이전 입력값 유지. FE
연령 범위 초과 (만 20~35세 밖) 가입 차단 + "곧 더 넓혀갈 거야" 안내 + 대기 리스트(waitlist) 등록 유도 FE + BE (waitlist 테이블)
탈퇴 후 같은 이메일로 재가입 30일 유예 기간 내: "아직 계정이 남아있어. 복구할래?" / 유예 종료 후: 새로 가입 BE (Edge Function)
닉네임 중복 실시간 중복 체크 → "이미 쓰고 있는 닉네임이야. 다른 걸 생각해봐" FE (API 호출) + BE (unique constraint)
이메일 인증 링크 만료 (24시간) "인증 링크가 만료됐어. 다시 보내줄게." + 재발송 버튼 FE
동시 다기기 로그인 허용 (Supabase 기본 동작). 로그아웃은 해당 기기만. BE (Supabase Auth 기본)
