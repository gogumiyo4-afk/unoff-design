UNOFF — 역대 최대 71.7만 공백기 청년 시장을 선점하는 6개월의 기회

작성일: 2026-03-12 | 전략기획팀
버전: v2.0 (첨삭 3단계 통합 반영본)
기반 문서: 브랜드 전략 통합요약, 시장·경쟁 분석, 첨삭 3건 (논리구조·설득력·데이터)

1. Executive Summary

71.7만 명이 쉬고 있는데, 그들을 위한 공간이 하나도 없다.

2025년 현재 20~30대 청년 71.7만 명이 아무것도 하지 않고 쉬고 있다. 통계청이 집계하기 시작한 이래 역대 최대 규모다.

이 중 72%인 51.6만 명은 자신이 선택한 것이 아니다. 구직 실패, 권고사직, 건강 문제로 밀려난 사람들이다. 그리고 그들의 83.5%가 "의욕 없다는 시선"을 경험한다.

문제는 이들이 갈 곳이 없다는 것이다.
- 블라인드는 "직장 인증 필요"
- 에브리타임은 "학교 인증 필요"
- 링크드인은 "커리어 있는 사람들의 공간"

쉬고 있는 사람은 어디에도 끼지 못한다.

UNOFF는 이 공백을 채운다.

OFF 상태를 정상화하는 커뮤니티. 비교 없이, 취업 강요 없이. 71.7만 명의 사람들이 처음으로 "나만이 아니다"를 확인하는 곳.

지금 당장 이 시장에 진입해야 하는 이유가 있다. 1년 이상 공백이 길어지면 복귀 의지가 90%에서 55%로 떨어진다. UNOFF가 없으면 51.6만 명 중 일부는 스스로 돌아오지 못하게 된다. 그리고 고용노동부는 지금 이 문제를 해결할 민간 파트너를 찾고 있다. 이 협업 윈도우는 2025년 하반기에 열려 있다.

6개월 내 행동하지 않으면 이 시장은 블라인드 또는 카카오의 카테고리로 편입된다.

2. 왜 커뮤니티인가, 왜 앱인가, 왜 지금인가

왜 커뮤니티 형식인가

  • 비자발적 공백층의 1순위 니즈는 "나만이 아니다"라는 확인 (고립감 해소)
  • 정보 제공(취업 공고)이 아닌 정서적 연결이 핵심 수요
  • 1:1 상담 서비스는 진입 장벽이 높다 → 익명 커뮤니티가 최저 마찰 형식

왜 앱인가

  • 웹 커뮤니티 대비 푸시 알림 기반 DAU 관리 가능
  • 배터리 게이미피케이션은 모바일 UX에 최적화
  • 에브리타임·블라인드 모두 앱 형식 → 타깃의 앱 커뮤니티 이용 습관이 존재

왜 지금인가

  • 71.7만 명 역대 최대 (2025년 통계청 경제활동인구조사) → 수요 정점
  • 고용노동부 파트너십 윈도우 (2025 하반기) → 6개월 내 착수해야 협의 가능
  • 블라인드 카테고리 확장 가능성 → 선점유율 확보 필요

3-1. 커뮤니티 기능

MVP — 게시판 카테고리 (5개)

카테고리 설명 브랜드 언어
오늘의 OFF 일상 공유, "오늘 어땠어?" 중심 자유게시판
면접후기 면접 경험 익명 공유, 공백 설명법 취업 정보
고민창고 고민 털어놓기, 비판 없는 공감 공간 정서 지원
같이 해요 스터디·프로젝트·소모임 모집 네트워킹
공지/가이드 운영진 공지, 커뮤니티 가이드라인 운영

MVP 제외 카테고리:
- 공동구매 → Phase 2 (초기 유저 규모로는 거래 성사율 낮음)
- 정부 지원사업 정보 → Phase 2 (큐레이션 인력 필요)

MVP — 익명/실명 하이브리드 시스템

기본값: 익명 ("OFF 중인 누군가")
선택적 공개:
  - 닉네임 사용 (나만의 정체성)
  - 뱃지 표시 (학교인증·공백 기간·관심 분야)
  - 실명 연동은 Phase 3 이후 (신중 접근)

설계 원칙: 스펙·학력 비교로 이어질 수 있는 정보는 기본값 비공개

MVP — ON/OFF 상태 표시 — "나 지금 충전 중이야"라는 선언

왜 필요한가: 공백기 청년의 가장 큰 심리적 장벽은 "나만 이러는 것 같다"는 고립감이다. ON/OFF 상태 표시는 "나도 OFF 중이에요"를 공개 선언하게 함으로써 이 고립감을 커뮤니티 연대감으로 전환하는 핵심 메커니즘이다. 이 기능이 없으면 UNOFF는 그냥 게시판이다.

상태값 (4가지):
  OFF   → 충전 중 (기본값, 공백 중)
  SLOW  → 배터리 올라오는 중 (복귀 준비)
  ON    → 다음 단계로 (취업·진학 완료)
  PAUSE → 잠깐 멈춤 (번아웃 등 일시 휴식)

표시 위치:
  - 프로필 상단 (배터리 아이콘 시각화)
  - 게시글 작성 시 상태 태그 자동 표시 (선택적)

주의:
  - ON 상태로 변경 시 → 탈퇴 유도 없음, "응원해요" 메시지만
  - ON 상태 유저도 커뮤니티 잔류 가능 (순환 구조 핵심)

MVP — 댓글/반응 시스템

기본 반응: 좋아요(하트) 대신 "같아요" (동일 경험 공감)
추가 반응:
  - "힘내요" (응원)
  - "나도 그랬어" (경험 공유)
  - "정보 있어요" (댓글 유도)

금지 반응:
  - 숫자 순위 (좋아요 수 상위 노출 없음)
  - "부럽다" 류 반응 제거 (비교 유발)

MVP — 콘텐츠 자생 루프 설계

커뮤니티 앱 최대 리스크는 "썰렁한 커뮤니티"다. 유저가 왜 계속 게시글을 올리는지의 자생 구조가 없으면 어떤 기능도 의미 없다.

자생 루프:
  씨앗 유저 50명 → 초기 게시글 50개 직접 작성 (운영팀)
  → 신규 유저 첫 방문 시 콘텐츠 밀도 확인
  → 공감 댓글("같아요") 받으면 배터리 충전 → 게시글 추가 작성 동기
  → 게시글 작성 유저가 다시 댓글 달기 유도
  → 댓글 작성도 배터리 충전 → 선순환

운영팀 콘텐츠 확보 계획:
  - 씨앗 유저 50명 × 주 2회 게시글 = 주 100건
  - 초기 3개월간 운영팀이 직접 톤세팅 콘텐츠 작성 병행
  - 게시글/일 100건 달성 시 운영팀 직접 작성 점진적 축소

3-2. 게이미피케이션 — 배터리 충전 시스템 — 비교 없는 유일한 성장 지표

왜 필요한가: 일반 커뮤니티의 "좋아요 수" "팔로워 수"는 비교를 유발하고 탈락감을 만든다. 공백기 청년에게 이 메커니즘은 독이다. 배터리 충전 시스템은 "어제의 나"와 비교하는 유일한 지표로 설계되어, 성취감은 주되 비교 압박은 없애는 유일한 게이미피케이션이다. 이 기능이 없으면 UNOFF는 차별화가 없다.

핵심 설계 원칙

비교가 아닌 절대치 기반. 남보다 높은 충전량이 아닌, 어제의 나보다 높은 충전량.

충전 포인트

활동 충전량 설계 근거
앱 접속 (일간) +5% 꾸준한 접속 습관 형성
게시글 작성 +10% 자기 표현 = 회복의 신호
댓글 달기 +5% 연결감 형성
"같아요" 반응 받기 +3% 공감받는 경험
면접후기 등록 +15% 고가치 콘텐츠 기여
프로필 완성 +20% (1회) 온보딩 완성
같이해요 모집 참여 +10% 오프라인 연결 시도
리부트 패키지 신청 +20% 전환 행동 시작

방전 설계:
- 7일 미접속 시 배터리 잔량 10% 감소
- "방전 주의" 알림: 3일 미접속 시 발송

충전 레벨 (5단계)

레벨 배터리 명칭 혜택
1 0~20% 방금 꺼짐 기본 기능 사용
2 21~40% 충전 시작 같이해요 모집 가능
3 41~60% 절반 충전 면접후기 열람 무제한
4 61~80% 거의 다 됐어 리부트 패키지 우선 매칭
5 81~100% 풀 충전 멘토 신청 가능

중요: 레벨 수치는 다른 유저와 비교 불가 — 내 프로필에서만 확인

뱃지 시스템 (Phase 2)

시간 기반 뱃지: "3일 연속 접속" / "한 달 버틴 사람" / "100일 충전 중"
활동 기반 뱃지: "면접 경험 공유자" / "프로젝트 완성" / "응원 10번 받음"
상태 전환 뱃지: "OFF → ON 전환 완료" (졸업 뱃지) / "다시 돌아온 사람" (재가입)

3-3. 매칭/네트워킹

MVP — "같은 상태" 매칭

매칭 기준 (3가지 조합):
  1. 공백 기간 (0~3개월 / 3~6개월 / 6개월~1년 / 1년 이상)
  2. 공백 이유 (구직 실패 / 번아웃 / 건강 / 기타)
  3. 관심 분야 (IT / 크리에이티브 / 비즈니스 / 학업 / 기타)

매칭 결과:
  - "나와 비슷한 OFF 중인 사람 3명"을 주 1회 추천
  - DM 먼저 보내는 것 가능, 거절도 쉽게

UI 언어: "지금 같은 배터리 잔량인 사람이에요"

Phase 2 — 프로젝트 팀 매칭

프로젝트 게시 형태:
  - 이름: "공백기 포트폴리오 프로젝트"
  - 기간: 1~3개월 / 인원: 2~5명
  - 모집 조건: OFF 중인 사람만

성과물: 완성 프로젝트는 포트폴리오 전환 가능, 앱 내 "포트폴리오 갤러리" 게재

3-4. 리부트 패키지

Phase 2 — 기업 연계 인턴십·교육 매칭

이 기능은 UNOFF의 수익 엔진이자, 유저에게 "원할 때" 복귀 경로를 제공하는 기능이다. 핵심 원칙: 취업 강요 없음. 리부트 패키지는 선택 옵션이며, 메인 피드에 취업 강요 콘텐츠가 없다.

수익 구조: B2B (기업이 UNOFF에 매칭 수수료 지불)

대상 기업:
  - 공백기 친화적 스타트업 (채용 공고에 "공백 무관" 명시)
  - 교육 기관 (부트캠프, 직업 훈련 기관)

매칭 플로우:
  1. 유저: 리부트 패키지 관심 표시 (선택, 강요 없음)
  2. UNOFF: 배터리 레벨 4+ 유저에게 맞춤 추천
  3. 기업: UNOFF 유저 풀에 접근 (유료)
  4. 유저: 클릭 → 기업 JD 확인 → 직접 지원

UI 주의:
  - 리부트 패키지 배너는 앱 메인이 아닌 별도 탭
  - "지금 취업 안 해도 됩니다"를 탭 진입 전 명시

Phase 2 — 정부 지원사업 큐레이션

데이터 소스: 고용노동부 API + 청년정책포털 크롤링
큐레이션 기준:
  - 공백기 친화적 (자격 요건에 경력 불필요)
  - 신청 마감 30일 이내
  - 월 5~10개 선별 노출

표시 언어: "지금 상태에서 바로 신청 가능한 것들이에요"

3-5. 안전망 설계

MVP — 커뮤니티 가이드라인

명시적 금지:
  - 스펙·연봉·학벌 자랑
  - "왜 아직도 취업 안 해요?" 형태의 압박 댓글
  - 다단계·홍보·광고성 게시물
  - 개인정보 수집 시도

권장:
  - 경험 공유 (정답 제시 아닌 경험 나누기)
  - 공감 댓글 (해결책보다 "나도 그랬어")
  - 익명 유지 존중

MVP — 비교·취업 강요 방지 메커니즘

설계 수준 차단:
  - 유저 간 배터리 레벨 비교 UI 없음
  - "취업했어요" 게시글은 별도 탭으로 분리 (메인 피드 노출 안 함)
  - 추천 알고리즘에 "높은 스펙 게시글 상위 노출" 없음

운영 수준 차단:
  - 비교·압박 댓글 → 운영진 경고
  - AI 모더레이션: 비교·압박 키워드 감지 후 검토 대기

MVP — 위기 상황 감지 및 상담 연결

감지 키워드 (예시): "살기 싫다", "포기하고 싶다", "아무 의미 없어"

감지 시 자동 응답:
  1. "잠깐, 지금 많이 힘드시죠?" 팝업
  2. 정신건강위기상담전화 1577-0199 / 자살예방상담전화 1393 연결
  3. 게시글 발행 전 확인 (선택)

운영 정책:
  - 자살·자해 관련 게시글은 즉시 비공개 처리
  - 운영팀 24시간 모니터링 (초기 운영팀 직접, 이후 AI+팀)

법적 검토 필수 항목 (출시 전 법무법인 또는 정신건강 전문가 자문 선 확보):
- 자살예방법 상 민간 기업의 온라인 위기 개입 범위
- 개인정보보호법 상 정신건강 관련 정보(민감 정보) 처리 동의 방식
- 의료법 상 심리 상담 연계 서비스 허용 범위

MVP — 신고·차단 시스템

신고 유형: 가이드라인 위반 / 스팸·광고 / 혐오·차별 / 위기 신호 감지

처리 프로세스:
  1. 신고 접수 → 24시간 내 검토
  2. 1회 위반: 경고 메시지
  3. 2회 위반: 7일 글쓰기 제한
  4. 3회 위반: 계정 영구 정지

4. 인증 및 가입 플로우

4-1. 가입 플로우 전체 (온보딩 7단계)

[Step 1] 간편 가입
  - 카카오 / 네이버 / 애플 로그인
  - 이메일 가입도 지원 (앱스토어 심사 대응)

[Step 2] 연령 인증 (20~35세)
  - 생년월일 입력 → 자동 계산
  - 범위 초과 시: "현재 우리가 집중하는 연령대예요.
    향후 확장 예정이니 조금만 기다려주세요." (거절이 아닌 대기 등록)

[Step 3] 상태 선택 (핵심 단계)
  "지금 어떤 상태인가요?" (강요 없이 선택)
  - 잠깐 쉬는 중 (3개월 미만)
  - 쉬고 있는 중 (3개월~1년)
  - 오래 쉬고 있어요 (1년 이상)
  - 번아웃 회복 중
  - 그냥 구경 중 (선택 안 해도 됨)

[Step 4] 공백 이유 선택 (선택 사항)
  구직 실패 / 퇴사·권고사직 / 건강 문제 / 자발적 휴식 / 말하기 어려워요

[Step 5] 관심 분야 선택 (2~3개)
  IT·개발 / 디자인·크리에이티브 / 경영·비즈니스 / 교육·연구 / 예술·문화 / 기타

[Step 6] 닉네임 설정
  - 기본값: "OFF 중인 누군가 #1234" (완전 익명)
  - 변경 가능 (단, 개인 식별 정보 포함 불가)

[Step 7] 첫 피드 진입 + 환영 메시지
  "지금 너랑 같은 상태인 사람이 [N]명 있어.
   여기서는 아무것도 증명 안 해도 돼."

5-3. Firebase vs Supabase 선택 근거

MAU 5,000명 기준 월 인프라 비용 비교:

Firebase (Blaze 종량제):
  - Firestore 읽기/쓰기: DAU 1,500명 기준 약 $30~80/월
  - Storage: 이미지 100GB 기준 $2.6/월
  - Functions: 호출 수 기준 약 $10~30/월
  - 합계 추정: $42~112/월

Supabase (Pro Plan):
  - Fixed: $25/월 (DB 8GB, 스토리지 100GB 포함)
  - 초과분: 별도 과금
  - 합계 추정: $25~50/월

결론: MAU 5,000명 기준 Supabase가 약 40~55% 저렴
      단, MAU 10만 명 이상에서는 Firebase가 더 안정적일 수 있음

추가 선택 이유: 매칭 기능(공백 기간×이유×관심 분야 교차 쿼리)이 핵심 기능이므로
              SQL 기반(Supabase)이 NoSQL(Firebase) 대비 개발 속도에 유리

[확인 필요: 실제 Firebase vs Supabase 벤치마크 테스트 — MVP 구축 단계에서 실측 비교 권고]

6. 수익 모델 상세

Phase별 수익 전략 개요

Phase 기간(목표) 수익 핵심 목표

Phase 1 — 무료 운영

  • 목적: 첫 1만 명 가입, 5,000 MAU, 커뮤니티 톤 확립
  • 비용: 인프라 $66~106/월 + 운영팀 인건비
  • 수익: 없음 (정부 파트너십 보조금 가능성 탐색)

Phase 2 — B2B 리부트 패키지 (3가지 시나리오)

수익 구조:
  기업 측 지불:
    - 채용 성공 수수료: 확정 채용 1건당 50~100만원
    - 부트캠프/교육기관: 수강 연결 1건당 5~20만원
    - 기업 프로필 등록 (연간 구독): 100만원/년

시나리오 기반 수익 예측 (MAU 1만 명 시점):

시나리오 관심률 전환율 건당 수수료 월 수익
보수 10% 5% 30만원 1,500만원
기본 10% 10% 30만원 3,000만원
낙관 15% 15% 50만원 1억 1,250만원

보수 시나리오만으로도 손익분기점 달성 가능 (인건비 5인 기준 월 1,500~2,500만원 수준)

전환율 가정 근거:
- 잡플래닛 기업 리뷰 → 채용 공고 클릭률: 약 8~12% (추정)
- 원티드 관심 표시 유저 대비 지원 전환율: 약 15~25%
- UNOFF 기본 시나리오 10%는 공백기 청년의 추가 심리적 장벽을 반영한 보수적 하향 조정치

[확인 필요: 채용 플랫폼 업계 공개 전환율 데이터]

Phase 3 — 다각화

광고:
  - 대상: 공백기 친화적 브랜드 (교육, 자기계발, 취미)
  - 금지: 취업 압박성 광고 (스카우트, 취업 강의 전면 광고)
  - 형식: 피드 인-피드 광고 (네이티브 형식)
  - 예상: MAU 5만 명 기준 CPM 3,000원 → 월 1,500만원

공동구매 수수료:
  - 대상: 공백기 관련 상품 (독서실 이용권, 자격증 교재, 심리 상담권)
  - 수수료: 거래액의 5~10%
  - 예상: 월 100건 × 평균 5만원 × 7.5% = 월 37만원 (초기)

프리미엄 기능 (검토 중):

     "공백기" "쉬는 청년" 키워드 검색 유입 = 250명 (추정)

9. 리스크 및 대응

9-1. 커뮤니티 톤 붕괴 시나리오 — 그리고 우리가 이미 막은 방법

커뮤니티 서비스 역사에서 가장 흔한 실패 패턴은 콘텐츠 변질이다.
(예: 뽐뿌가 정보 공유 → 자랑으로 변질, 에브리타임 취업 게시판의 압박 분위기)

UNOFF는 이 패턴을 설계 수준에서 차단한다:
1. 취업 성공 게시글 = 메인 피드 노출 없음 (별도 "ON 전환" 탭 분리)
2. 스펙 비교 기능 없음 (알고리즘에서 비교 가중치 제거)
3. 씨앗 유저 50명이 초기 톤을 세팅 (설계된 문화 이식)

이 중 하나라도 빠지면 3개월 내 커뮤니티 분위기가 바뀐다. 세 가지 모두 설계에 이미 반영되어 있다.

9-2. 유저 이탈 — 취업 후 순환 구조

리스크: 핵심 타깃이 취업하면 앱을 떠나는 구조적 이탈

대응 (순환 구조 설계):

ON 전환 유저 → "이제 ON이 됐네요. 돌아올 자리는 항상 있어요." 메시지
→ 멘토 역할 제안 (배터리 레벨 5 혜택으로 연결)

멘토 인센티브 (구체화):
  1. "UNOFF 졸업 멘토" 뱃지 (커리어 포트폴리오 가치)
  2. 리부트 패키지 파트너 기업 소개 우선권 (역방향: 멘토가 채용담당자 역할)
  3. 멘토 전용 네트워킹 이벤트 초대

→ OFF 중인 후배와 매칭 기회 제공
→ "나중에 다시 OFF 올 수도 있잖아요" 잔류 유도

순환 지표 목표: ON 전환 유저의 20%가 6개월 내 멘토로 전환

이 인센티브가 없으면 ON 전환 = 앱 이탈의 등식이 고착된다.

9-3. 심리적 안전 이슈

리스크: 위기 상황 유저의 자해·자살 관련 게시글 → 잘못된 대응 시 법적 책임 및 브랜드 손상

대응:

- AI 위기 감지 시스템 (OpenAI Moderation + 커스텀 키워드)
- 정신건강복지센터·자살예방상담전화 1393 연결 시스템
- 운영팀 위기 대응 매뉴얼 작성 (Mental Health First Aid 기반)
- 정신건강 전문가 자문단 1명 이상 확보 권고

법적 검토: 출시 전 법무법인 자문 필수 (의료법·자살예방법·개인정보보호법)

9-4. 규제 리스크

개인정보:
  - 공백 이유·상태 등 민감 정보 수집 → 개인정보보호법 준수 필수
  - 권고: 최소 수집 원칙 + 암호화 저장 + 동의 절차 강화

청소년 접근:
  - 20세 미만 접근 차단 (생년월일 인증)
  - Phase 2에서 본인인증 API 도입 검토

광고/수수료:
  - 앱스토어 인앱결제 수수료 15~30% 발생
  - 권고: 외부 웹사이트로 연결하는 방식으로 우회 설계

10. 예상 반론 및 선제 대응

예상 반론 대응
"취업 앱이랑 뭐가 달라요?" 취업을 목표로 설계하지 않습니다. 리부트 패키지는 선택 옵션이며, 메인 피드에 취업 강요 콘텐츠가 없습니다.
"유저가 취업하면 다 떠나지 않나요?" ON 전환 유저의 멘토 전환 설계로 순환 구조를 확보합니다. 취업 후 이탈이 아닌, 취업 후 멘토가 되는 구조입니다.
"오픈카톡이랑 뭐가 달라요?" 오픈카톡은 분산·휘발적이며 브랜드가 없습니다. UNOFF는 배터리 게이미피케이션·구조화 콘텐츠·안전망 설계로 차별화합니다.
"시장이 작지 않나요?" 71.7만 명은 에브리타임 MAU의 절반 수준입니다. 이 숫자는 구조적으로 줄어들지 않습니다.
"수익화가 너무 늦지 않나요?" 6개월 파일럿 수익화는 조기입니다. 커뮤니티 앱은 MAU 1만 명 전에 수익화하면 신뢰를 잃습니다.
