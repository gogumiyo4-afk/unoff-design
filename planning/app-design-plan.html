이 문서의 목적
기존 통합기획서(5_최종_통합기획서.md)가 "왜, 무엇을" 다루었다면,
이 문서는 "정확히 어떤 화면이 어떻게 동작하는가"를 다룹니다.
화면별 와이어프레임, UX 카피, 데이터 모델, 에러 처리까지 — 이 문서만으로 개발 착수가 가능합니다.

에러 처리

상황에러 메시지대응
네트워크 없음"인터넷 연결을 확인해줘"재시도 버튼
소셜 로그인 취소(무동작 — 원래 화면 유지)—
다른 소셜로 가입한 이메일"이 이메일은 [카카오]로 가입되어 있어"해당 소셜 버튼 하이라이트
이메일 미인증"아직 이메일 인증이 안 됐어"재발송 버튼
비밀번호 틀림"비밀번호가 맞지 않아"비밀번호 찾기 링크
서버 오류"잠깐 문제가 생겼어. 조금 뒤에 다시 시도해줘"재시도 버튼

세션 관리

JWT 토큰 저장: expo-secure-store (SecureStore)
• access_token: 1시간 유효 (Supabase 기본) / refresh_token: 30일 유효
• 앱 시작 시 refresh_token 확인 → 유효하면 자동 갱신 → 만료 시 LoginScreen
• 로그아웃: 토큰 삭제 + signOut() → LoginScreen

설계 원칙
• 같은 게시글에 여러 반응 가능 (같아요 + 힘내요 동시)
• 같은 반응 재탭 = 취소
• 제거된 반응: "부럽다", 숫자 랭킹 (비교 유발 방지)
• 반응 수는 표시하되 정렬에 과도한 영향 없음

공구 플로우

1. 공구 등록 (Lv.4+ 또는 운영팀) → 운영팀 승인
↓
2. 참여 신청 (Lv.3+) → 실시간 참여자 수 표시
↓
3. 최소 인원 도달 → 결제 (외부 링크/카카오페이)

브랜드 오리진 스토리 & 톤 원칙

UNOFF의 두 겹 — 표면은 가볍게, 속뜻은 깊게. 톤은 위로가 아닌 리스펙트.

줄임말의 묘미: 처음엔 UNOFF라는 짧은 한 단어로 접근하지만,
그 속뜻이 Underdog's Offbeat임을 알게 되었을 때 유저들이 느끼는 브랜드의 무게감이 커진다.

주의: 앱 내에서 "언더독"을 전면 노출하지 않음. "언더독 = 싸워서 이기는 사람" 뉘앙스가 비교 없는 설계와 충돌할 수 있음.

톤 원칙: 위로 → 리스펙트

기존 — 위로

"괜찮아, 쉬어도 돼"
"힘내, 금방 취업할 거야"

업그레이드 — 리스펙트

"네 박자는 틀린 게 아니라 독특한 거야"

왜 리스펙트인가?
위로는 결국 "약자에게 하는 것"이라는 뉘앙스가 남는다. 타깃 유저가 가장 싫어하는 것이 동정이다. 리스펙트는 대등한 관계에서 나온다.
